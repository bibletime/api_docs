48 /**
49  * @page backend The structure of the backend
50  * <p>
51  * The backend implementation for Sword is called CSwordBackend, the classes we use
52  * to work with keys are called CSwordVerseKey and CSwordLDKey, both are derived from
53  * the class CSwordKey.
54  * The CSwordKey derived classes used for Sword do also inherit the classes
55  * VerseKey (CSwordVerseKey)
56  * and SWKey (CSwordLDKey).
57  * </p>
58  * <p>
59  * The classes used to handle all module based stuff are derived from CSwordModuleInfo.
60  * The module classes are: CSwordModuleInfo (for Sword modules),
61  * CSwordBibleModuleInfo (for bibles), CSwordCommentaryModuleInfo (for commentaries) and
62  * CSwordLexiconModuleInfo (for lexicons).
63  * Have a look at the class documentation of the mentioned classes to learn how the
64  * structure of them looks like and which class inherits which other class.
65  * </p>
66  * <p>
67  * The first objects which should be created in the application is the backend
68  * (for Sword the class is called CSwordBackend).
69  * Then create all the different module classes for the correct Sword modules.
70  * Have a look at
71  * BibleTime::initBackend() to see how it's done in BibleTime.@br
72  * Later you can work with them for example by using the CSwordKey and
73  * CSwordModuleInfo derived class.
74  * </p>
75  */

77 /**
78  * @page frontend The structure of the frontend
79  *
80  * <p>
81  * The frontend contains the classes which interact with the user. For example the
82  * display windows, the searchdialog or the other parts.
83  * </p><p>
84  * The display windows which provide functionality are CBibleReadWindow for
85  * Bibles, CBookReadWindow for books, CCommentaryReadWindow for commentaries and CLexiconReadWindow for
86  * lexicon and dictionaries. CHTMLWriteWindow and CPlainWriteWindows are used for editing the Personal Commentary.
87  * </p><p>
88  *
89  * The class CDisplayWindow is the class that various views with in the windows are derived.
90  * </p><p>
91  *
92  * Another important part of the frontend are the keychoosers.
93  * They provide an interface to choose a key of a module.
94  * The interface for different module types is different.
95  * The base class is CKeyChooser which is the factory for the derived classes.
96  * Use the function CKeyChooser::createInstance to get the correct
97  * keychooser implementation for the desired module.<br/>
98  * </p><p>
99  * Some major toolbar widgets are CKeyChooser and BtDisplaySettingsButton.
100  * </p>
101  */

103 /** @mainpage BibleTime - Source Code Documentation
104  * BibleTime main page.
105  * <p>This is the source code documentation of BibleTime, a Bible study tool.
106  * <p>
107  * The main class of BibleTime is called @ref BibleTime, which is the main window
108  * and initializes all important parts at startup.
109  * </p><p>
110  * BibleTime is divided in two major parts, the backend and the frontend.

113  * to allow easy access to it's functionality and to have it in a (more or less :)
